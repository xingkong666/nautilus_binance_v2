# configs/strategies/

## Purpose
Per-strategy YAML configuration files consumed by `AppFactory` when loading strategies at runtime. Each file maps a strategy name to its Python class path, indicator parameters, instrument list, bar subscription type, and lifecycle settings. The `vegas_ema_combo_*.yaml` variants represent tuned parameter sets generated by `scripts/param_sweep.py` for different capital phases and market regimes.

## For AI Agents

### Working In This Directory
- Each YAML must have a `strategy.class_path` resolvable via `src.strategy.<module>:<Class>`.
- `bar_type` format: `"{instrument_id}-{period}-{aggregation}-LAST-INTERNAL@{external_period}-{external_agg}-EXTERNAL"`.
- `instruments` is a list of NautilusTrader instrument IDs (e.g., `BTCUSDT-PERP.BINANCE`).
- To add a new strategy config: copy the nearest existing YAML, change `name` and `class_path`, adjust params.
- Run a quick backtest to verify: `uv run python scripts/run_backtest.py --config configs/strategies/<new>.yaml --env configs/env/dev.yaml`.

### Common Patterns
```yaml
strategy:
  name: my_strategy
  class_path: strategy.my_module:MyStrategy
  params:
    fast_ema_period: 10
    atr_period: 14
  instruments:
    - BTCUSDT-PERP.BINANCE
  bar_type: "BTCUSDT-PERP.BINANCE-15-MINUTE-LAST-INTERNAL@1-MINUTE-EXTERNAL"
  close_positions_on_stop: true
  reduce_only_on_stop: true
```
- `capital_pct_per_trade` takes precedence over `trade_size` when both are present.
- `signal_cooldown_bars` prevents re-entry for N bars after a signal.

## Dependencies

### Internal
- `src.app.factory.AppFactory` — loads and instantiates strategies from these files
- `src.strategy.*` — strategy classes referenced by `class_path`
- `src.core.config.load_app_config` — merges strategy YAML into `AppConfig`

### External
- `pydantic` — `BaseStrategyConfig` validation (frozen `StrategyConfig`)
- `nautilus_trader` — instrument IDs, bar type parsing
