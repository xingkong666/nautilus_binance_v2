# experiments/

## Purpose
Research and validation artifacts generated by backtesting, walk-forward analysis, parameter sweeps, and grid searches. This directory is the output sink for `src/backtest/` — it is never read by the live trading system. Contents are used to select strategy parameters, validate robustness across time periods and instruments, and produce the reports referenced in `docs/`.

## For AI Agents

### Working In This Directory
- **Read-only from the live system**: nothing in `experiments/` is imported by `src/`. Modifying files here has no effect on live or backtest execution.
- **Each walk-forward window** contains a `train/` and `test/` subdirectory plus named strategy-variant subdirs. The `train/` split is used for parameter fitting; `test/` is the OOS evaluation held out during fitting.
- **Naming convention for walk-forward experiments**: `vegas_ema_combo_<variant>` where the variant suffix describes the key differentiating factor (instrument set, regime filter, gate relaxation, grid scoring).
- **`param_sweep/` best-JSON files** are the canonical parameter outputs to copy into `configs/strategies/` when promoting a result to production. Always verify OOS metrics before promoting.
- **`reports/` are UUID-named**: use `walkforward_summary.csv` or `walkforward_aggregate.json` from the relevant walk-forward directory to identify which report UUID corresponds to which run.
- **`sweep/` is transient**: do not treat its contents as stable; it may be overwritten by the next sweep run.
- **Do not delete `walkforward/` subdirs** without archiving — they are the evidence base for parameter choices documented in `docs/strategy_200pct_blueprint_2026_03.md` and `docs/vegas_tunnel_rollout_2026_03.md`.

### Common Patterns
- **Run a parameter sweep**:
  ```bash
  uv run python scripts/run_param_sweep.py \
    --strategy vegas_tunnel \
    --config configs/strategies/vegas_tunnel.yaml \
    --output experiments/param_sweep/
  ```
- **Run walk-forward validation**:
  ```bash
  uv run python scripts/run_backtest.py \
    --config configs/strategies/vegas_tunnel.yaml \
    --env configs/env/dev.yaml \
    --walkforward --windows 6 \
    --output experiments/walkforward/my_new_run/
  ```
- **Review aggregate walk-forward results**:
  ```python
  import pandas as pd
  df = pd.read_csv("experiments/walkforward/vegas_ema_combo_multi_refined/walkforward_summary.csv")
  print(df[["window", "sharpe", "max_drawdown", "total_pnl"]])
  ```
- **Promote best params to production config**:
  ```bash
  # Review OOS metrics first, then copy:
  cp experiments/param_sweep/vegas_tunnel_BTCUSDT_best.json \
     configs/strategies/vegas_tunnel_prod_params.json
  ```

## Dependencies

### Internal
- `../src/backtest/runner.py` — `BacktestRunner` writes report subdirs to `reports/` and summary CSVs to `walkforward/`
- `../src/backtest/walk_forward.py` — orchestrates window splits, writes `walkforward/<experiment>/window_*/`
- `../src/backtest/regime.py` — regime-filtered experiments (`*_best_regime*`) depend on this module
- `../scripts/run_backtest.py` — CLI entry point that populates this directory
- `../scripts/run_param_sweep.py` — generates `param_sweep/` outputs
- `../data/processed/catalog/` — `BacktestRunner` reads bar data from here; experiments are invalid if catalog data changes

### External
- `pandas` / `pyarrow` — summary CSV and Parquet I/O
- NautilusTrader backtest engine — drives all simulation runs that produce these outputs
